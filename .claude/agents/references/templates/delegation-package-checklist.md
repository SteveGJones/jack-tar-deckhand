# Delegation Package Checklist

This checklist keeps persona WHAT artefacts and SOP HOW artefacts bundled before they enter the [Chapter 9](../chapters/Chapter-09-Realizing-AI-First-Service-Architecture.md) release gate. Complete one row per persona and store alongside the readiness scorecard.

| Item | Description | Owner | Status | Evidence |
| --- | --- | --- | --- | --- |
| Persona template | Final template with approvals and telemetry fields. | Service owner |  | Link to file |
| SOP summary + YAML | Table + YAML snippet stored per [Appendix G](../appendices/Appendix-G-SOP-Integration-Patterns.md). | SOP owner |  | Link to repo |
| Cross-domain register row | Borrowers + dependency IDs logged. | AI RM steward |  | Register entry |
| Vanilla-Agent log | Latest dry run recorded with pass/fail + issues. | AI RM lab lead |  | Log ID |
| MCP deployment plan | Endpoint, version, rollback plan documented. | Platform ops |  | Runbook |
| Readiness scorecard | All checkpoints green; Change Advisory Circle minutes attached. | Facilitator |  | Scorecard |
| Backlog linkage | Work item IDs for persona, SOP, and integration tasks. | Delivery manager |  | Jira/Azure ID |
| Funding confirmation | Opex/capex approvals for persona + SOP operations. | Finance partner |  | Approval doc |

> **Usage:** Treat the checklist as the cover sheet for each delegation package folder referenced in Chapters 6, 8, and 9.
