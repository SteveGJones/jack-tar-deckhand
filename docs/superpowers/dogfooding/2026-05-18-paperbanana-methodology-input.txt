The Jack-Tar Deckhand system is a Claude Code presentation engineering pipeline that produces conference-quality PowerPoint decks from structured TalkBriefs. The system is distributed as five plugins via the Claude Code marketplace, with one orchestrator plugin coordinating four specialised engine plugins plus an optional external CLI tool.

The orchestrator plugin (jack-tar-deckhand) sits at the centre of the system. It exposes eleven user-facing skills covering the full pipeline lifecycle: brand-manager (extract or load corporate brand profiles), slide-stylist (derive palette, typography, and layout rules), narrative-architect (build slide outlines with narrative arcs), strategy-map (classify per-slide rendering strategies), smartart-selector (recommend SmartArt graphic types), smartart-extractor (transform slide content into engine-specific structured data), speaker-notes-writer (generate timed speaker notes), imagegen-bridge (route image generation to available providers), deck-assembler (assemble the final .pptx via PptxGenJS or python-pptx), deck-qa (run thirty anti-pattern checks), and verify (report pipeline capability and engine readiness). It also defines six AI persona agents that participate in specific pipeline stages: deck-conductor (the top-level workflow orchestrator), prompt-engineer (composes image generation prompts), image-reviewer (visual quality gate that returns compact JSON verdicts), presentation-reviewer (reviews assembled decks against conference best practices), vision-analyst (analyses generated images for backdrop positioning), and smartart-selector (proposes graphic types in negotiation with narrative-architect).

Four engine plugins specialise in distinct rendering capabilities and are coordinated by jack-tar-deckhand at run time. The jack-tar-ollama plugin provides local-tier image generation via Ollama models running on the operator's machine and serves draft renders during iteration. The jack-tar-cloud plugin provides cloud-tier image generation across multiple providers including Google Gemini (Nano Banana Flash and Pro tiers, Imagen Fast and Standard), OpenAI GPT Image, FAL FLUX, and Recraft V4 and serves production renders. The jack-tar-msft-smartart plugin renders editable PowerPoint SmartArt graphics across twenty-nine layouts derived from the open Open-XML-SDK fixtures so the resulting graphics remain editable in PowerPoint after delivery. The jack-tar-custom-smartart plugin renders custom data-visualisation graphics via Mermaid CLI, Vega-Lite CLI, and matplotlib for charts, diagrams, and infographics that fall outside SmartArt's native catalogue.

A sixth peer plugin, jack-tar-superpower-bridge, wraps the orchestrator to enrich outputs from the upstream slash-pptx skill in the superpowers-toolkit plugin. The bridge offers a two-phase flow: a narrative pre-brief that captures communication intent before slash-pptx runs, and a transactional enrichment pass that layers Jack-Tar visuals onto an existing slash-pptx-produced deck without modifying the original.

An optional external CLI tool, paperbanana, is invoked via subprocess for publication-tier academic figures when slides match the academic_figure rendering strategy. Paperbanana is a sibling orchestrator running its own multi-agent pipeline (Retriever, Planner, Stylist, Visualizer, Critic) over Google Gemini. When paperbanana is absent, the imagegen-bridge falls back to Nano Banana Flash with academic-figure-aware prompting.
