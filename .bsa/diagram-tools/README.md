# Service Architecture Diagram Tools

This directory contains the **official diagram renderer** for the AI-First Architecture Toolkit.

## ⚠️ CRITICAL: service_architecture_renderer.py is the ONLY approved renderer

**USE THIS:** `service_architecture_renderer.py`
- Generates professionally styled SVG diagrams
- Inter font family, shadows, business-friendly appearance
- Consistent brand identity from `design/style-config.json`
- Suitable for executive presentations and documentation
- Uses Graphviz **for layout computation only** (`-Tjson`), then generates custom styled SVG

**DO NOT:**
- ❌ Generate raw DOT files and call `dot -Tsvg` directly
- ❌ Use `subprocess.run(['dot', '-Tsvg'])` in custom scripts
- ❌ Create alternative renderers that bypass this tool

Raw Graphviz output produces technical diagrams with automatic styling that are unsuitable for business stakeholders.

## Usage

### Via the toolkit skill (recommended):
```bash
/service-architecture-renderer
```

### Direct script invocation:

**Enterprise L0 view:**
```bash
python3 .bsa/diagram-tools/service_architecture_renderer.py \
  canonical-model.json --l0 output.svg
```

**Domain drill-down (e.g., Manufacturing L1):**
```bash
python3 .bsa/diagram-tools/service_architecture_renderer.py \
  canonical-model.json mfg 1 manufacturing-l1.svg
```

## Prerequisites

- Python 3.8+
- Graphviz (optional but recommended): `brew install graphviz`
- Style config at `design/style-config.json` (auto-discovered)

## Why Not Raw Graphviz?

| Approach | Output Quality | Use Case |
|----------|----------------|----------|
| **service_architecture_renderer.py** (correct) | Professional styled SVG | Business presentations, documentation, methodology work |
| Raw `dot -Tsvg` (incorrect) | Technical Graphviz defaults | Never - produces unsuitable output |

## How It Works

1. **Layout Computation**: Calls `dot -Tjson` to get node positions and edge paths
2. **Custom SVG Generation**: Takes layout data and generates styled SVG in Python
3. **Brand Identity**: Applies styling from `design/style-config.json`
4. **Professional Output**: Inter font, shadows, clean appearance

This two-step process gives us professional business-quality diagrams while leveraging Graphviz's excellent layout algorithms.

## Output Examples

- ✅ **Correct**: Professional SVG with Inter font, shadows, clean styling
- ❌ **Wrong**: Raw Graphviz SVG with `<!-- Generated by graphviz -->` comments and default styling

If your diagram contains `<!-- Generated by graphviz -->` comments or looks technical rather than business-friendly, you're using the wrong approach.
